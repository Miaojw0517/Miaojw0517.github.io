从 push rejected 电脑死机后，git命令引发的各种报错问题!

发表于2022-06-22|更新于2022-12-07|Git

感慨下

事发昨天上午，可真是郁闷的一上午，搞的手忙脚乱的，看到git 操作报各种错误，有一种想让我跑路的感觉😂，悲痛不已，以此作为笔记，张张记性，下次遇到，给自己留个后手！

push rejected 原因

本地分支 与 远程分支 文件冲突！可能在我 拉代码的时候 没拉到，以为是最新的，这时候本地分支 合并了自己的代码，便要 push 的时候，远程分支的内容已被其他同事更新，此刻我在提交我本地代码的时候可能出现了与远程分支文件冲突，被rejected了，因此才会被提示需要重新pull远程的分支至本地分支！正常情况下 应该重新pull完新的代码后，在和我自己的代码 重新 merge下，在 push提交上去应该就可以了！

问题流程

一切缘由从 push rejected 代码提交被拒绝开始, 一个陌生的警告，让我开始有点慌了，因此我点开了 show details 查看具体问题，大概意思是让我重新pull下远程分支的代码后，merge 自己的新代码并重新push代码，然后我按照操作，重新pull了下文件，拉下来’8’个新的文件，重点来了，当我重新merge 和 push的时候，电脑突然就死机了，发出了鸣叫……，😰，迫不得已，强制将电脑关机了，后续。。。电脑重启，重新Git操作，开始各种报错问题，此时此刻一脸懵逼，没有思绪，遍开始 各种 百度寻找问题的解决方案，但最终结果，只要涉及到Git命令操作的都会报错形式展示，无奈又无助，没有系统学习过git，遇到问题遍是如此。。。难耐！不过，后面，百度还是找到有用的解决方案，内容的描述，即大概意思是，在Git操作时，断电，或特殊情况(强制关机)导致被中断，影响到了.git 隐藏文件夹里的 分支管理文件出了问题，且将对应的报错(本地以及远程)分支文件删除后，重新 将 远程分支 checkout下到本地即可！

处理方案

注：这是找到解决方案的开始，将对应报错的分支文件删除后，在重新（删除的是本地分支，别把远程的干掉!）delete branch 把原有的分支删除就能成功了，若不删除，此删除，便会报错，导致无法删除，删除完后针对远程orgin/civil-dev分支重新进行checkout，此时操作会报错，依然需要把 .git下的 orgin 下的对应报错分支删除,重新checkout即可!

简书：详解 push rejected 诱发的错误描述


报错信息：unable to resolve reference ‘refs/remotes/origin/feature/dev_init‘: reference broken

问题：突然断电导致git pull失败，提示上面内容（文件损坏） 解决方法：删除.git文件夹下的（上面路径的文件）refs/remotes/origin/feature/dev_init，再pull就好了
